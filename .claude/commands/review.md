---
description: コミット前のコードレビューチェックリストを実行します
---

コミット前のコードレビューを実施してください。以下の項目をチェックしてください：

## 1. コード品質

- [ ] TypeScriptの型エラーがない
- [ ] `any` 型を使用していない
- [ ] 未使用のimportや変数がない
- [ ] Console.logなどのデバッグコードが残っていない

## 2. コーディング規約

- [ ] Biomeのルールに準拠している（`pnpm lint`）
- [ ] ファイル命名規則に従っている
- [ ] コンポーネントは関数宣言で定義している
- [ ] JSDocコメントが適切に記載されている

## 3. 機能性

- [ ] エラーハンドリングが適切に実装されている
- [ ] エッジケースを考慮している
- [ ] バリデーションが適切に行われている
- [ ] Server/Client Componentsが適切に分離されている

## 4. セキュリティ

- [ ] ユーザー入力が適切にバリデーションされている
- [ ] 機密情報（APIキー等）がコードに含まれていない
- [ ] XSS/SQL Injection対策が考慮されている

## 5. パフォーマンス

- [ ] 不要な再レンダリングがない
- [ ] 重い処理が適切に最適化されている
- [ ] N+1クエリ問題がない

## 6. テスト

- [ ] 必要なユニットテストが追加されている
- [ ] すべてのテストがパスする（`pnpm test`）
- [ ] エッジケースのテストがある

実際にコードをチェックし、問題があれば具体的に指摘してください。
すべて問題なければ「✅ レビュー完了。コミット可能です」と報告してください。
