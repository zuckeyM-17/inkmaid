---
description: 新機能のテストケースを提案します
---

最近追加した機能または指定された機能について、包括的なテストケースを提案してください。

## テストケースの観点

### 1. 正常系テスト

- 基本的な動作が正しく機能する
- 期待される入力に対して正しい出力が得られる
- UIが適切に更新される

### 2. 異常系テスト

- 不正な入力に対するバリデーション
- エラーハンドリングが適切に動作する
- エラーメッセージが適切に表示される

### 3. エッジケース

- 境界値のテスト（空文字列、最大値、最小値）
- null/undefined の処理
- 非同期処理のタイミング

### 4. 統合テスト

- 他の機能との連携
- データの整合性
- パフォーマンス

## テスト実装

各テストケースについて、Vitestを使用したテストコードを提案してください。

```typescript
import { describe, it, expect } from "vitest"

describe("機能名", () => {
  it("テストケース説明", () => {
    // テストコード
  })
})
```

テストコードは具体的で実行可能なものを提供してください。
