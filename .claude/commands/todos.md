プロジェクト内のTODOを確認して整理します。

## 1. doc/TODO.mdの表示

まず、`doc/TODO.md`ファイルの内容を読み取り、以下の情報を表示してください：

- 🔴 優先度高のタスク
- 🟡 中優先度のタスク
- 🟢 最近完了したタスク（直近5件程度）

## 2. コード内のTODOコメントの検索

次に、コードベース内のTODOコメントを検索してください：

- `apps/web/src/**/*` 内の以下のパターンを検索：
  - `// TODO:`
  - `/* TODO:`
  - `// FIXME:`
  - `// HACK:`
  - `// XXX:`

検索結果は以下の形式で整理：

```
ファイルパス:行番号
  TODO内容
```

node_modules, dist, .next, build などのビルド成果物は除外してください。

## 3. サマリーの作成

最後に、以下のサマリーを作成してください：

- 📋 TODO総数
- 🔴 優先度高のタスク数
- 🟡 中優先度のタスク数
- 💬 コード内のTODOコメント数

---

見やすく整理して、次に何をすべきかが明確になるように表示してください。
