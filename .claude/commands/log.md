---
description: 開発ログを自動生成します
---

作業終了時に必要なドキュメントをすべて更新してください。

## 1. 開発ログの作成

今日の作業内容を振り返り、`doc/logs/YYYY/MM/YYYYMMDD-N.md` 形式で開発ログを作成してください。

以下の情報を含めてください：

1. **実装内容の概要**
   - 追加した機能や修正したバグ
   - 主な変更点

2. **追加・変更したファイル**
   - ファイルパスと変更内容のリスト
   - 重要な変更は詳細を記載

3. **技術的な詳細**
   - 実装で工夫した点
   - 直面した課題と解決方法
   - 今後の改善余地

4. **次のステップ**
   - 残りのTODO
   - 今後の実装予定

## 2. API リファレンスの更新

以下の変更を行った場合は `doc/api-reference.md` を更新してください：

- tRPCルーターの新規追加・変更
- REST APIエンドポイントの追加
- 型定義の変更
- 使用例のコード追加

## 3. TODO リストの更新

機能を完了した場合は `doc/TODO.md` を更新してください：

- 完了したタスクを「🟢 完了済み」セクションに移動
- 新しいタスクがあれば優先度に応じて追加
- 「最終更新」の日付を更新

---

最新のgit diffを確認し、実際の変更内容に基づいてドキュメントを作成してください。
ログファイルのパスは今日の日付を使用し、既に同じ日のログがある場合は番号をインクリメントしてください。
